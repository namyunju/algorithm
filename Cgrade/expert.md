# 삼성 Expert 시험을 준비하며

## 기계적 최적화
제약 조건 내에서 성능을 극대화하는 설계 및 운영 기술

**시간 비용**
함수의 입력값이 커짐에 따라 알고리즘 비용이 얼마나 빠르게 증가하는가.

### 검색 알고리즘
- 선형 검색 (O(n))
정렬되지 않은 테이블에서 사용 가능. 정렬되어 있다면 O(n)이지만 평균 약 2배 빠름

- 이진 검색 (O(log2n))
성능 뛰어나지만 가장 빠른 건 아님. 입력 데이터가 키를 기준으로 정렬되어 있어야 함.

- 보간 검색 (O(loglogn))
이진 검색과 같이 정렬된 테이블을 두 부분으로 나눔.

- 해싱 (O(1))
키를 해시 테이블의 배열 색인으로 변환. 별도의 해시 테이블이 필요.최악의 경우 O(n) 성능.

### 최적화 패턴
- 사전 계산
- 지연 계산
- 배칭
- 캐싱
- 특수화
- 더 큰 조각 선택하기
- 힌팅
- 예상 경로 최적화
- 해싱
- 이중 검사

## 구조체 

### 기본 구조체
```cpp
// 구조체 정의 (타입 정의. 메모리 할당은 X)
struct Student {
    int id;
    char name[50];
    double gpa;
};

// 구조체 변수 선언 (스택에 메모리 할당됨)
Student s1;
// 메모리상 상태
// s1 => [id: ?] [name: ??] [gpa: ?]
```

### 구조체 멤버 접근 : .연산자
```cpp

Student s1;
s1.id = 202401;
strcpy(s1.name, "윤주");
s1.gpa = 4.5;

printf("ID: %d\n", s1.id);
```

### 구조체 배열
여러 학생을 저장하고 싶을 때
```cpp
Student students[10];
students[0].id = 202401;
strcpy(students[0].name, "윤주");
students[0].gpa = 4.5;
```

### 구조체 포인터 : 동적 할당
```cpp
//포인터 변수 선언
Student *ptr;
//동적할당
ptr = (Student *)malloc(sizeof(Student));

// 메모리상 상태:
// 힙에 새로운 메모리 할당됨
// 주소 1000번지 : [id][name][gpa]
// ptr => [주소값:1000]

ptr->id = 202403;
strcpy(ptr->name, "김철수");
ptr->gpa = 3.9

printf("ID: %d\n", ptr->id);

// 메모리 해제
free(ptr); // 힙에서 메모리 반환
ptr = NULL; // 포인터 초기화
```
